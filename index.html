MDshow

以 Markdown 标签页为核心的 AI 文档工作台

把导入、阅读、编辑、翻译、AI 对话与导出放进同一个工作环境，让研究材料、公文草稿、知识笔记和交付文件围绕一份可长期维护的 Markdown 母稿流转。

产品定位

让内容创作重新站到前台

MDshow 1.0 已经不只是轻量 Markdown 编辑器。它更像一个文档处理平台，把外部资料转换成可继续加工的 Markdown，再让阅读、修改、翻译、AI 协作和最终导出保持在同一个标签页世界里。

它的思路很直接：Markdown 承载内容，标签页组织工作过程，AI 辅助格式、结构、翻译和对话。格式劳动退到后台，用户把注意力留给判断、组织和表达。

主工作流

从资料入口到交付出口，一条链走完

  1. 01

    导入外部材料

    接收 Word、Excel、PowerPoint、PDF、网页、图片和音频，把它们转换成适合继续处理的 Markdown 工作材料。

  2. 02

    阅读与编辑

    在渲染视图、源码视图、双栏视图之间切换，保持长文阅读焦点，继续清洗、摘录、改写和组织。

  3. 03

    翻译与 AI 协作

    按标签页隔离翻译状态，把 AI 对话写入同一份文档，也可以通过文档助手审阅和受控改写。

  4. 04

    导出交付文件

    把当前 Markdown 母稿导出为 Word、PDF、HTML、Excel 或 PowerPoint，让交付格式成为工作流的出口，而不是唯一源文件。

核心能力

MDshow 1.0 已经形成的工作闭环

Tabs

标签页工作台

多标签并行承载来源、草稿、译文、AI 输出和交付中间稿。每个标签页保留独立内容、视图状态、翻译状态和撤销历史。

Views

三种文档视图

渲染视图适合阅读，源码视图适合直接改写，双栏视图适合核对。视图切换会尽量保持用户正在看的位置。

Vim

实用 Vim 工作流

支持常用移动、删除、复制、粘贴、撤销、重做、可视选择和命令模式，也支持通过命令导出常见交付格式。

AI

对话即文档

AI 对话不是孤立弹窗，而是标签页内容的一部分。对话记录、上下文和生成结果都能沉淀为 Markdown 工作记录。

Project

项目工作区

打开目录作为项目后，可以使用项目树、作用域助手、项目知识面板和写作 runtime，适合跨多份材料的长链路写作。

Safety

文档安全与恢复

自动保存、未保存提示、关闭保护、崩溃恢复和标签切换安全共同降低长时间写作中的数据风险。

10 分钟快速上手

第一次打开 MDshow 时，可以从空白窗口走到第一份可用输出：

  1. 按 Cmd+N 新建空白文档，新的空白文档会直接进入可编辑源码视图。
  2. 用 Cmd+O 打开已有 Markdown 文件，已有文档会先进入渲染视图，适合先阅读。
  3. 把来源、草稿和输出分别放进不同标签页，让每份材料都有清楚的位置。
  4. 内容成熟后，用导出功能生成 Word、PDF、HTML、Excel 或 PowerPoint 文件。

打开和导入的区别

如果 .md 已经是主工作文件，用打开继续直接编辑；如果输入来自 .docx、.xlsx、.pptx、.pdf、网页、图片或音频，用导入转换成 Markdown 工作材料。

导入路径会尽量保留正文、表格、脚注、批注、备注、图片和结构线索，并通过 warning、阶段信息或统计数据解释结果来自确定性解析、AI 增强、OCR 还是回退路径。

阅读、编辑、标签页与分屏

MDshow 提供渲染、源码和双栏三种视图。渲染适合浏览长文，源码适合精确编辑，双栏适合对照核查。多标签和同文档分屏让来源材料、草稿和 AI 输出可以并排处理。

Vim 层是效率工具，覆盖常用移动、编辑和命令模式。用户仍然可以结合鼠标选择、macOS 菜单和快捷键完成更快的操作。

翻译、AI 对话与文档助手

翻译会话按标签页隔离。配置 Gemini、Qwen、OpenAI 或 DeepSeek 后，可以渐进翻译、中途停止，或把译文导出成新标签页。

AI 对话标签适合把提问和回答保留成工作记录。文档助手则面向受控改写，会先准备变更预览，再由用户决定接受、拒绝或回退。

项目工作区与安全边界

通过 Project 菜单打开目录后，MDshow 会显示项目树、记住最近项目，并把项目根目录作为助手和 runtime trace 的参考边界。

项目助手、文档上下文、文件操作和写作 runtime 都被设计成显式、可控、可追踪，尤其适合跨多份源资料、提纲文件、工作笔记和交付产物的长文工作。
